# Interview Project Script

## Quick Start

- Read `references/script.md` for narrative structure and Q&A strategy.
- Read `references/baseline.md` to enforce truthfulness and language rules.

## Inputs Checklist

- Project name, role, and time range
- Resume bullets or whitepaper notes
- Key pain points, constraints, decisions, and trade-offs
- Metrics or qualitative results
- Legacy assets (docs, standards, components)
- Evidence anchors (design docs, PRs/commits, dashboards) for key claims
- Target level (P5/P6/P7) if applicable

## Workflow

1. Draft the 60-second hook using business scale, responsibility, pain, solution, and result.
2. Expand STAR-Plus with concrete details, decisions, and obstacles.
3. Add defensive Q&A: trade-off challenges, under-the-hood questions, and deficiencies.
4. Simulate 3 interview rounds and refine weak areas.

## Output Format

Provide:

- Hook (1 minute)
- STAR-Plus deep dive (S/T/A/R/L sections)
- Defensive Q&A set (trade-off, fundamentals, reflection)
- Three-round mock interview prompts

## Language Requirements

- All script content must be written in Chinese.

## Guardrails

- No invented metrics or claims.
- Tie every technical detail to business impact.
- Keep language precise and interview-friendly.
