# 代码审查报告模板规范

## 定位

本规范定义结构化报告格式，用于呈现审查结果（来自 Oracle/子代理或人工审查）。

## 报告结构

固定 4 个检查项：

1. **需求实现完整性**：需求点全部覆盖且实现一致
2. **线上风险评估**：稳定性/性能/安全/一致性风险
3. **变更范围控制**：变更是否仅限需求相关
4. **代码规范符合性**：是否遵循规范与架构约定

## 输出规则

- 每个检查项：✅ 通过 或 ❌ 不通过
- 问题列表格式：`文件:行` - 影响 - 建议
- 风险类问题必须标注严重级别：【高/中/低】
- 无问题明确写"无问题"或"未发现问题"

## 报告模板

```markdown
# 代码审查报告 - 分支：[分支名]

**审查范围**：从基准分支分叉后的所有提交（[提交范围]）
**审查时间**：[日期]

## 审查结果概览

| 检查项 | 结果 | 说明 |
| --- | --- | --- |
| 需求实现完整性 | ✅/❌ | [一句话] |
| 线上风险评估 | ✅/❌ | [一句话] |
| 变更范围控制 | ✅/❌ | [一句话] |
| 代码规范符合性 | ✅/❌ | [一句话] |

## 1. 需求实现完整性检查

**结果**：✅ / ❌
- 问题1（可选）：`路径:行` - 影响 - 建议

## 2. 线上风险评估

**结果**：✅ / ❌
- 【高/中/低】问题1（可选）：`路径:行` - 影响 - 建议

## 3. 变更范围控制

**结果**：✅ / ❌
- 问题1（可选）：`路径` - 影响 - 建议

## 4. 代码规范符合性

**结果**：✅ / ❌
- 问题1（可选）：`路径:行` - 影响 - 建议

## 总体建议

[一句话优先级建议；无问题写"无"]

## 复核记录（可选）

- 条目完整性：✅/❌（文件/行/影响/建议齐全）
- 严重级别：✅/❌（风险项已标注）
- 结论一致性：✅/❌（结论与问题列表一致）
```

## 复核要点（可选）

如果包含复核记录，确保：
1. 问题条目具备文件/行、影响、建议三要素
2. 风险类问题标注严重级别（高/中/低）
3. 总体建议与各检查项结果一致
