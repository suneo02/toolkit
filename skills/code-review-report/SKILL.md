# Code Review Report

## 概览

用最小字数输出中文审查报告，聚焦问题、风险、缺失与建议；无问题则明确说明。

## 流程

1) 确认范围：基准分支、提交范围、需求或 spec。
2) 阅读变更：定位关键逻辑、数据流与边界处理。
3) 形成问题清单：仅列出问题，按严重度排序；每条包含文件/行、影响与建议。
4) 按模板输出：严格使用模板，避免额外解释。

## 缺失信息提示

- 基准分支与提交范围是什么？
- 是否有需求文档或 spec 可以对齐？
- 是否只输出报告正文（Markdown）？
