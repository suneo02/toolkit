# 归档模板与规则速查

## 完整模板

```markdown
# [对话主题]

**创建时间**：YYYY-MM-DD HH:MM
**主要参与者**：用户
**核心目标**：[1-2 句说明对话目的]
**预计产出**：[文档/代码/方案等]

---

## 1 YYYY-MM-DD HH:MM 本次对话【模型】

**💬 用户输入**：
[完整记录用户提问或指令]

**🤖 本次响应处理**：

- **分析要点**：[…]
- **执行动作**：[…]
- **产出内容**：[…]
- **注意事项**：[…]

**📊 关键结论/成果**：
[1-3 条要点]

**🔗 相关上下文**：
[与历史记录的关联或后续提示]

---

📝 **对话记录已更新**

**文件状态**：新建 → `文件名.md`
**本次记录摘要**：[1 句总结]
**下次延续提示**：如需继续此话题，请提及“关键词”
```

## 对话类型判定（参考）

- **默认新建**：用户未明确要求“继续/追加/延续”，且未提供目标文件路径。
- **追加**：用户明确要求“继续/追加/延续”，或提供了已有文件路径。
- **仅在必要时提问**：用户要求追加但未提供目标文件时，询问一次目标文件；否则不询问。

## 追加规范（关键）

- 追加前必须读取目标文件完整内容。
- 将新记录拼接到末尾后整体写回，禁止直接覆盖写入。

## 语言要求

- 记录文本必须中文，即使用户输入为英文也需中文描述。

## 新建文件头

```markdown
# [对话主题]

**创建时间**：YYYY-MM-DD HH:MM
**主要参与者**：[用户角色/姓名]
**核心目标**：[1-2句话说明对话目的]
**预计产出**：[期望得到的成果，如文档、代码、方案等]

---
```

## 记录条目

```markdown
## [序号] [时间戳YYYY-MM-DD HH:MM] [会话标签]【当前使用的AI模型】

**💬 用户输入**：
[完整记录用户的提问或指令，保留原始语气和关键细节]

**🤖 本次响应处理**：

- **分析要点**：[解读用户需求的核心要点]
- **执行动作**：[具体采取了哪些行动]
- **产出内容**：[生成了什么具体成果，尽量量化]
- **注意事项**：[需要特别关注的点或限制条件]

**📊 关键结论/成果**：
[本次对话中最重要的1-3个结论或产出物]

**🔗 相关上下文**：
[与历史记录的关联点，或对后续讨论的提示]
```

## 结尾提示

```
---

📝 **对话记录已更新**

**文件状态**：[新建/追加] → `文件名.md`
**本次记录摘要**：[1句话总结记录内容]
**下次延续提示**：如需继续此话题，请提及"[关键词]"
```

## 场景扩展字段（可选）

- 技术开发讨论：可增加“技术栈”“代码变更”“问题状态”等字段。
- 文档撰写协助：可增加“文档类型”“目标读者”“完成进度”等字段。

## 命名规则

- 仅允许使用字母、数字、下划线 `_` 与连字符 `-`。
- 默认：`dialog_<YYYYMMDD_HHMM>.md`；可选主题前缀：`<topic>_dialog_<YYYYMMDD_HHMM>.md`
